SERVER
    service = tudo que é regra de negócio ou processamento 
    controller = intermediar a camada de apresentação e a camada de negócio
    routes = camada de apresentação
    server = responsácel por criar o servidor (mas não instancia-lo)
    index = intanciar o servidor e expõe para a web (infra)
    config = tudo o que for estático do projeto
